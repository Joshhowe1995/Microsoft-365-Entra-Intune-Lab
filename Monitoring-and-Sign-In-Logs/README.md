Monitoring & Sign-In Logs

Goal
Demonstrate the ability to monitor authentication activity and interpret Conditional Access decisions using Microsoft Entra sign-in and audit logs.

What This Demonstrates

Visibility into user sign-in activity

Identification of Conditional Access outcomes (allowed or blocked)

Understanding of how policies are evaluated during authentication

Use of audit logs to validate access decisions and policy behaviour

Evidence
See the Evidence folder for screenshots showing:

Entra sign-in logs with Conditional Access results

Audit logs confirming policy evaluation or configuration activity

Core Competencies Demonstrated

Log analysis

Troubleshooting access issues

Operational monitoring

Security awareness and validation
