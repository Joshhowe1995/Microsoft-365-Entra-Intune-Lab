# Conditional Access with Device Compliance

## Goal
Demonstrate Zero Trust access control by enforcing Conditional Access based on device compliance state.

## What This Shows
- Access decisions depend on both identity and device health
- Conditional Access evaluates device compliance in real time
- Non-compliant devices are blocked from access

## Evidence
- Conditional Access policy requiring a compliant device
- Sign-in result showing access allowed or blocked based on compliance

## Core Competencies Demonstrated
- Conditional Access policy design
- Device compliance as an access signal
- Identity + device + access integration
- Real enforcement validation
